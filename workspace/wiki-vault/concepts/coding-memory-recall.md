# Coding memory recall

## Summary

Coding memory recall is the practice of retrieving the smallest useful working context before starting implementation or debugging work.

It exists to reduce repeated rediscovery after session restart, compaction, or project switching.

## Target working set

A good coding recall pass should recover:

- current project or repo
- recent goal
- key files or components
- decisions already made
- blocker, if one exists
- next step

## Operational rule

For coding tasks, prefer targeted recall from recent memory and transcripts before broad repository search.

## Why it matters

This keeps the agent from wasting turns on rediscovering context that was already known in a previous session.
